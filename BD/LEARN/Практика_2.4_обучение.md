# Практика №2.4 — обучение (дата/время, оконные функции, интервалы)

Ниже — синтаксис, который нужен, чтобы понимать решения по практике 2.4.

## 1) Дата, время, timestamp

- `DATE` — только дата.
- `TIME` — только время.
- `TIMESTAMP` — дата + время.

Примеры литералов:

```sql
'2023-03-13'::DATE
TIME '09:00:00'
TIMESTAMP '2023-03-13 09:00:00'
```

В отчёте используется сокращённый вид для `TIMESTAMP` (PostgreSQL сам распознаёт):

```sql
'2023-03-13 09:00:00'
```

## 2) Приведение типов `::`

- `event_ts::DATE` — взять дату из `TIMESTAMP`.
- `event_ts::TIME` — взять время из `TIMESTAMP`.

## 3) Интервалы и вычитание времени

В PostgreSQL можно вычитать `timestamp` и получать `interval`:

```sql
next_ts - event_ts
```

## 4) Перевод интервала в секунды: `EXTRACT(EPOCH FROM interval)`

`EXTRACT(EPOCH FROM ...)` возвращает длительность в секундах (как число):

```sql
EXTRACT(EPOCH FROM (next_ts - event_ts))
```

Чтобы получить часы:

```sql
EXTRACT(EPOCH FROM (next_ts - event_ts)) / 3600.0
```

## 5) Оконные функции: `LEAD(...) OVER (...)`

Оконные функции позволяют смотреть «следующую» строку без процедурного цикла.

В отчёте:

```sql
LEAD(event_ts) OVER (PARTITION BY employee_id ORDER BY event_ts) AS next_ts
LEAD(reader_value) OVER (PARTITION BY employee_id ORDER BY event_ts) AS next_val
```

- `PARTITION BY employee_id` — отдельно по каждому сотруднику.
- `ORDER BY event_ts` — порядок событий.
- `LEAD` — взять значения из следующей строки в этом порядке.

## 6) Как считается отработанное время из событий входа/выхода

События:

- `reader_value = 1` — вход
- `reader_value = 2` — выход

Логика: для каждой пары «вход (1) → следующий выход (2)» суммировать разницу времени:

```sql
CASE
  WHEN reader_value = 1 AND next_val = 2 THEN EXTRACT(EPOCH FROM (next_ts - event_ts))
  ELSE 0
END
```

## 7) `COALESCE` и `GREATEST`

- `COALESCE(x, y)` — если `x` равен `NULL`, взять `y`.
- `GREATEST(a, b)` — максимум из значений (удобно для «не меньше 0»).

Пример «штраф только если опоздал»:

```sql
GREATEST(0, minutes_late)
```

## 8) Сбор «меток дня» под расчёт опозданий

В отчёте для каждой даты берутся ключевые моменты:

- первый вход (`first_in`)
- уход на обед (`lunch_out`)
- возвращение (`lunch_in`)
- последний выход (`last_out`)

Это сделано подзапросами `SELECT MIN(...)` / `SELECT MAX(...)` с фильтрами по дате и типу события.

## 9) Математика для коэффициента A

Условие:

- если нет опозданий → \(A=1\)
- за каждые 10 минут опоздания \(A\) уменьшается на 0.05
- \(A \ge 0\)

Шаги:

1) Посчитать общие минуты опоздания `total_late_minutes`.
2) Посчитать число «десятков минут»: `FLOOR(total_late_minutes/10.0)`.
3) Применить штраф: `1 - 0.05 * tens`.
4) Ограничить снизу: `GREATEST(0, ...)`.

## 10) Округление

Чтобы красиво выдавать деньги/часы:

```sql
ROUND(value::NUMERIC, 2)
```
