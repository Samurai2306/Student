# Практика №2.1 — обучение (PostgreSQL + PL/pgSQL)

Ниже — минимально необходимый синтаксис, чтобы понимать код из отчёта по практике 2.1.

## 1) Типы данных, которые используются

- `DATE`: дата без времени.
- `TIMESTAMP`: дата и время.
- `INTEGER`: целое число.
- `DECIMAL(10,2)`: десятичное число с точностью (10 цифр всего, 2 после точки).
- `TEXT`: строка.
- `BOOLEAN`: `TRUE` / `FALSE`.

## 2) Создание функций: `CREATE OR REPLACE FUNCTION`

Общий вид:

```sql
CREATE OR REPLACE FUNCTION имя(параметры...)
RETURNS тип_результата
LANGUAGE plpgsql
AS $$
BEGIN
  -- тело
  RETURN ...;
END;
$$;
```

- `OR REPLACE` позволяет пересоздавать функцию без ручного удаления.
- `LANGUAGE plpgsql` включает процедурный язык (можно делать `IF`, циклы, переменные).
- Блок `$${ ... }$$` — текст тела функции (dollar-quoting).

## 3) `NULL` и проверки `IS NULL`

- `NULL` — «нет значения».
- Сравнивать с `NULL` через `= NULL` нельзя; нужно `IS NULL` / `IS NOT NULL`.

Пример:

```sql
IF birth_date IS NULL THEN
  RETURN NULL;
END IF;
```

## 4) Возраст: `AGE()` и `EXTRACT()`

- `AGE(date1, date2)` возвращает интервал времени между датами.
- `EXTRACT(YEAR FROM интервал)` вытаскивает число лет.

Пример:

```sql
EXTRACT(YEAR FROM AGE(CURRENT_DATE, birth_date))::INT
```

`::INT` — приведение типа (cast).

## 5) Ветвления: `IF / ELSIF / ELSE`

Пример:

```sql
IF x < 0 THEN
  ...
ELSIF x < 10 THEN
  ...
ELSE
  ...
END IF;
```

## 6) Исключения: `RAISE EXCEPTION` и `EXCEPTION`

### 6.1) Явно выбросить ошибку

```sql
RAISE EXCEPTION 'message %', value USING ERRCODE = '22003';
```

- `%` — место для подстановки аргумента.
- `ERRCODE` — SQLSTATE код ошибки (можно опустить, но полезно для диагностики).

### 6.2) Поймать ошибку внутри блока

```sql
BEGIN
  -- опасные операции
EXCEPTION
  WHEN OTHERS THEN
    -- обработка любой ошибки
END;
```

Важно: в PostgreSQL функция выполняется внутри транзакции вызвавшего запроса. Внутренний `BEGIN ... EXCEPTION ... END` — это подблок, который позволяет «перехватить» ошибку и продолжить выполнение, но не создаёт отдельную транзакцию `COMMIT/ROLLBACK`.

## 7) Возврат таблицы: `RETURNS TABLE` и `RETURN NEXT`

Если функция возвращает набор строк:

```sql
RETURNS TABLE(order_number TEXT)
```

Внутри можно делать:

```sql
order_number := '...';
RETURN NEXT;
```

Это добавляет одну строку в результирующий набор.

## 8) Переменные: `DECLARE` и присваивание `:=`

```sql
DECLARE
  i INTEGER;
  yyyymm TEXT;
BEGIN
  yyyymm := TO_CHAR(CURRENT_DATE, 'YYYYMM');
END;
```

## 9) Цикл `FOR`

```sql
FOR i IN 1..count LOOP
  ...
END LOOP;
```

## 10) Строки, конкатенация и форматирование

- Конкатенация: `'||'`
- Преобразование даты в строку: `TO_CHAR(date, 'YYYYMM')`
- Дополнение нулями слева: `LPAD(text, len, '0')`

## 11) Случайные числа: `RANDOM()`

- `RANDOM()` → число `double precision` от 0 до 1.
- Пример получения 0..999999:

```sql
FLOOR(RANDOM() * 1000000)
```

## 12) Таблицы и блокировки: `CREATE TABLE`, `SELECT ... FOR UPDATE`

### 12.1) Таблицы

```sql
CREATE TABLE IF NOT EXISTS accounts (
  id SERIAL PRIMARY KEY,
  balance DECIMAL(10,2) NOT NULL DEFAULT 0
);
```

- `SERIAL` — автоинкремент (в современных версиях можно `GENERATED ... AS IDENTITY`, но `SERIAL` часто используется в учебных).
- `DEFAULT` — значение по умолчанию.
- `NOT NULL` — запрет `NULL`.

### 12.2) Проверка существования и блокировка строки

```sql
SELECT balance INTO from_balance
FROM accounts
WHERE id = from_account_id
FOR UPDATE;
```

- `INTO переменная` записывает результат запроса в переменную.
- `FOR UPDATE` блокирует найденную строку до конца транзакции, чтобы конкурирующие переводы не испортили баланс.
- `FOUND` / `NOT FOUND` — системный флаг, был ли найден результат после `SELECT INTO` или `PERFORM`.

## 13) `PERFORM`, `UPDATE`, `INSERT`

- `PERFORM` выполняет запрос и игнорирует результат (удобно для проверки существования).
- `UPDATE ... SET ... WHERE ...` изменяет строки.
- `INSERT INTO ... VALUES ...` добавляет строку.
