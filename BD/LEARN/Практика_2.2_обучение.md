# Практика №2.2 — обучение (SQL-функции + агрегации)

Ниже — синтаксис и идеи, которые нужны, чтобы понимать функции из отчёта по практике 2.2.

## 0) Важная оговорка про таблицы

В PDF для 2.2 не приведена структура БД, поэтому в отчёте функции написаны под **типовую** схему:

- `properties(id, address, district, rooms, area_m2, price_rub, property_type, created_at, total_area_m2, ...)`
- `sales(id, property_id, realtor_id, sale_date, sale_price_rub, commission_rub, ...)`
- `realtors(id, last_name, ...)`

Если у тебя в БД другие названия таблиц/полей — нужно переименовать их в запросах 1:1.

## 1) `NUMERIC` vs `DECIMAL`

В PostgreSQL `DECIMAL` — это псевдоним `NUMERIC`. Оба подходят для денег (точная десятичная арифметика).

## 2) Функции на PL/pgSQL vs SQL

### 2.1) PL/pgSQL (`LANGUAGE plpgsql`)

Нужен, когда есть переменные, проверки, `IF`, обработка ошибок.

Ключевые элементы:

- `DECLARE ...` — объявление переменных
- `SELECT ... INTO var` — положить значение в переменную
- `IF ... THEN ... END IF` — ветвления
- `RAISE EXCEPTION` — ошибка

### 2.2) SQL-функции (`LANGUAGE sql`)

Если всё делается одним SQL-запросом (CTE, агрегации, фильтрация), проще писать так:

```sql
CREATE OR REPLACE FUNCTION f(...)
RETURNS TABLE(...)
LANGUAGE sql
AS $$
  SELECT ...
$$;
```

## 3) Даты: `EXTRACT(YEAR/MONTH FROM ...)`

Чтобы отфильтровать продажи по месяцу/году:

```sql
EXTRACT(YEAR FROM sale_date)::INT = year_num
EXTRACT(MONTH FROM sale_date)::INT = month_num
```

Приведение `::INT` делается, чтобы сравнение было «целое с целым».

## 4) Агрегации: `SUM`, `COUNT`, `AVG`, `GROUP BY`

- `SUM(x)` — сумма
- `COUNT(*)` — количество строк
- `AVG(x)` — среднее
- `GROUP BY column` — группировать строки по значению колонки

### `COALESCE`

Если `SUM(...)` возвращает `NULL` (нет строк), можно заменить на 0:

```sql
COALESCE(SUM(x), 0)
```

## 5) CTE (`WITH ... AS (...)`)

`WITH` позволяет разбить большой запрос на этапы, как на «временные таблицы»:

```sql
WITH base AS (...),
avg_by_district AS (...)
SELECT ...
```

## 6) Деление без ошибки: `NULLIF`

Чтобы не делить на ноль:

```sql
price_rub / NULLIF(area_m2, 0)
```

Если `area_m2 = 0`, `NULLIF(...,0)` вернёт `NULL`, и результат станет `NULL` вместо ошибки.

## 7) Поиск по строке: `ILIKE` и конкатенация

- `ILIKE` — регистронезависимый `LIKE`.
- Конкатенация строк: `'||'`.

Пример «подстрока входит»:

```sql
address ILIKE '%' || p_address || '%'
```

## 8) Проценты от общего: `CROSS JOIN` с итогом

Частый паттерн:

1) Посчитать агрегаты по типам.
2) Посчитать общий итог.
3) Присоединить итог ко всем строкам через `CROSS JOIN` (итог — одна строка).

```sql
WITH agg AS (...),
tot AS (SELECT SUM(sales_count) AS total_count FROM agg)
SELECT ...
FROM agg
CROSS JOIN tot;
```

## 9) Аннуитетная ипотека: `POWER`

Формула аннуитетного платежа:

\[
P = L \cdot \frac{r(1+r)^n}{(1+r)^n - 1}
\]

где:
- \(L\) — сумма кредита (`principal`)
- \(r\) — месячная ставка (`annual_rate/12/100`)
- \(n\) — число месяцев (`years*12`)

В SQL это:

```sql
principal * (r_month * POWER(1 + r_month, months)) / (POWER(1 + r_month, months) - 1)
```

И отдельный случай `r_month = 0` (без процентов): `principal / months`.
