# Все способы запустить функции практик 2.1–2.4 (PostgreSQL)

Ниже — **все основные способы**, как выполнить SQL из отчётов и **как вызвать каждую функцию**.

## 0) Важно про зависимости (чтобы не было `relation ... does not exist`)

- **Практика 2.1**: функции 1–3 не требуют таблиц; задание 4 требует таблиц `accounts`, `transactions` (в отчёте они создаются).
- **Практика 2.2–2.3**: требуют предметные таблицы (`properties`, `sales`, `realtors`, …). Если в вашей базе их нет, сначала создайте их/адаптируйте под вашу схему.
- **Практика 2.4**: самодостаточна (создаёт `рабочий_день` и данные).

## 1) Способ №1 — через `psql` (самый надёжный)

### 1.1) Подключиться к базе

```sql
SELECT current_database();
```

Если не та база:

```psql
\c имя_бд
```

### 1.2) Запустить целый файл отчёта

В `psql`:

```psql
\i c:/Users/user/Documents/Student/BD/Практика_2.1_отчёт.md
```

Важно: `\i` выполняет **SQL**, поэтому удобнее запускать `.sql`. Если файл `.md`, то лучше:
- либо копировать SQL-блоки вручную,
- либо сохранить/сделать отдельную копию отчёта в `.sql`.

Практический вариант: создать рядом `.sql` и вставить туда содержимое SQL-блоков из отчёта.

### 1.3) Запуск «кусочками»

В `psql` можно просто вставить SQL-блок из отчёта и выполнить (закончить `;`).

### 1.4) Полезные настройки `psql`

```psql
\set ON_ERROR_STOP on
\timing
```

## 2) Способ №2 — через любой SQL‑редактор (pgAdmin / DBeaver / DataGrip / Cursor SQL)

1) Открыть подключение к PostgreSQL.
2) Открыть Query Tool / Console.
3) Вставить SQL-блоки из отчёта.
4) Выполнить (обычно кнопка Run / Execute).

Важно: многие IDE выполняют **только выделенный текст**. Если ничего не выделять — выполняют весь скрипт.

## 3) Способ №3 — одним “мастер‑скриптом”

Если хотите запускать всё одной командой:
1) Создайте файл `run_all_practices.sql`
2) Скопируйте туда SQL из отчётов в правильном порядке:
   - Практика 2.1
   - (DDL предметных таблиц для 2.2–2.3, если нужно)
   - Практика 2.2
   - Практика 2.3
   - Практика 2.4
3) Выполните:

```psql
\i c:/Users/user/Documents/Student/BD/run_all_practices.sql
```

## 4) Как вызвать все функции (готовые команды)

Ниже команды **вызова** (их выполняют уже после `CREATE OR REPLACE FUNCTION ...`).

### 4.1) Практика 2.1 — вызовы

#### Задание 1 — `calculate_age`

```sql
SELECT calculate_age(DATE '2000-01-15');
SELECT calculate_age(NULL);
```

#### Задание 2 — `categorize_order`

```sql
SELECT categorize_order(999.99);
SELECT categorize_order(1000.00);
SELECT categorize_order(5000.00);
SELECT categorize_order(20000.00);
SELECT categorize_order(NULL);
```

#### Задание 3 — `generate_order_numbers`

```sql
SELECT * FROM generate_order_numbers(5);
SELECT * FROM generate_order_numbers(0);
```

#### Задание 4 — `transfer_money`

Перед вызовом добавьте тестовые счета:

```sql
INSERT INTO accounts(balance) VALUES (1000), (0) RETURNING *;
```

Вызов:

```sql
SELECT transfer_money(1, 2, 250.00) AS ok;
SELECT * FROM accounts ORDER BY id;
SELECT * FROM transactions ORDER BY id;
```

### 4.2) Практика 2.2 — вызовы

> Эти функции заработают только если у вас есть таблицы `properties`, `sales`, `realtors` и нужные поля.

#### Задание 2 — `realtor_month_salary`

#### Задание 5 — `sales_stats_by_year`

### 4.3) Практика 2.3 — вызовы

## 5) Проверка, что функция создалась

```sql
SELECT proname
FROM pg_proc
WHERE proname IN (
  'calculate_age',
  'categorize_order',
  'generate_order_numbers',
  'transfer_money',
  'convert_property_price',
  'realtor_month_salary',
  'mortgage_monthly_payment',
  'properties_below_district_avg',
  'sales_stats_by_year',
  'apply_property_discounts',
  'find_sales_date_anomalies',
  'check_property_area_consistency',
  'recalc_sales_commission',
  'week_work_hours',
  'calc_salary_with_tardiness'
)
ORDER BY proname;
```
