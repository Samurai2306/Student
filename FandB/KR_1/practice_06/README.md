## Как протестировать

### Практика 1 (CSS-препроцессоры)
- Откройте `practice_01/index.html` в браузере.
- Убедитесь, что карточка товара отображается с фото, названием и описанием.
- При необходимости скомпилируйте SASS: `cd practice_01 && npx sass styles.scss styles.css`

### Практика 2 (Express API)
- В папке `practice_02`: `npm install && npm start`
- В Postman или браузере: GET http://localhost:3000/products
- Проверьте POST, PATCH, DELETE по маршрутам из методички.

### Практика 3 (Postman и внешние API)
- Протестируйте API из ПЗ 2 не менее чем 3 запросами в Postman.
- Выберите внешнее API, выполните не менее 5 запросов.
- Скриншоты сохраните в `practice_03/screenshots/` и приложите к отчёту.

### Практика 4 (React + Express)
- **Сервер:** `practice_04/server` → `npm install && npm start` (порт 3000)
- **Клиент:** `practice_04/client` → `npm install && npm run dev` (порт 3001)
- Откройте http://localhost:3001: список товаров, фильтр по категории, сортировка, поиск, кнопка «Добавить товар», редактирование и удаление из карточки.
- Проверьте: создание товара с некорректной ценой — должно показать сообщение об ошибке с сервера.

### Практика 5 (Swagger)
- **Сервер:** `practice_05/server` → `npm install && npm start`
- Откройте http://localhost:3000/api-docs — интерактивная документация API.
- Выполните: GET /api/products (Try it out → Execute); POST /api/products с телом из примера; при желании проверьте PATCH и DELETE.
- Опционально: запустите клиент из `practice_04/client` при работающем сервере ПЗ 5 — интерфейс должен работать с этим API.

## Типичные проблемы и решения

| Проблема | Решение |
|----------|---------|
| Порт 3000 или 3001 уже занят | Закройте другое приложение на этом порту или смените порт в коде (например, в `vite.config.js` для клиента — `server.port`) |
| CORS: «blocked by CORS policy» при запросах с клиента ПЗ 4 | Убедитесь, что запущен именно сервер из `practice_04/server` (или ПЗ 5), в котором включён CORS для origin http://localhost:3001 |
| Страница клиента белая, в консоли ошибка сети | Сначала запустите сервер (practice_04/server или practice_05/server), затем обновите страницу клиента |
| Swagger UI не открывается | Проверьте, что сервер ПЗ 5 запущен и вы переходите по адресу http://localhost:3000/api-docs (порт тот, на котором запущен сервер) |
| В ПЗ 1 карточка без стилей | Откройте именно `index.html` из папки practice_01; убедитесь, что рядом есть файл `styles.css` (при необходимости выполните `npx sass styles.scss styles.css`) |
